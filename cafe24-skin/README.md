# 금빛농원 카페24 풀커스텀 스킨 (1차 MVP 골격)

설계서 v1.0 (`docs/design-spec.md` 또는 첨부 문서) 기준으로 만든 카페24 스마트디자인 스킨 골격이다. 카페24 어드민에 업로드하기 전의 작업 트리이며, 모듈/변수 영역은 카페24 표준 자리에 placeholder로 비워두었다.

## 폴더 구조

```
cafe24-skin/
├── layout/
│   ├── layout.html            # 공통 레이아웃 (헤더+푸터+모바일 플로팅바)
│   ├── _header.html           # import 대상: 상단 GNB
│   └── _footer.html           # import 대상: 푸터(법적 표기)
├── main/
│   └── index.html             # 홈 (§4.2 — 8개 섹션)
├── product/
│   ├── list.html              # 카테고리 (§4.3)
│   └── detail.html            # 상품 상세 (§4.4)
├── board/
│   ├── list.html              # 공지·후기·1:1문의 목록
│   └── view.html              # 게시글 보기
├── myshop/                    # 마이쇼핑 (카페24 기본 사용)
├── member/                    # 회원가입·로그인 (카페24 기본 사용)
├── order/                     # 장바구니·주문서·결제 (카페24 기본 사용)
├── css/
│   ├── tokens.css             # 브랜드 토큰 (색·간격·타이포 변수)
│   └── style.css              # 공통 스타일
├── js/
│   └── script.js              # GNB·플로팅·탭 등 인터랙션
└── SkinImg/                   # 스킨 전용 이미지
```

## 카페24 스마트디자인 문법 메모

- 리소스 연결: `<!--@css(/css/style.css)-->`, `<!--@js(/js/script.js)-->`
- 부분 포함: `<!--@import("/layout/_header.html")-->`
- 모듈 블록: `<div module="product_listmain_1">…##product_name##…</div>`
- 변수: `##변수명##` — 모듈이 노출하는 변수만 사용
- 캐시 우회: `style.css?ver=YYYYMMDD`

> 본 골격에서 `<!-- module: ... -->` 주석으로 표시된 영역은 카페24 어드민에서 정확한 모듈ID를 부여한 뒤 실제 모듈 변수를 채워 넣어야 한다 (1.2절). 임의 마크업으로 대체하면 결제·장바구니가 깨질 수 있다.

## 디자인 토큰 출처

`css/tokens.css` — 설계서 §2.1·§2.2·§2.3·§2.4 그대로 매핑.

## 콘텐츠 표현 주의 (설계서 §7)

- 질병명·치료·면역 보장 표현 금지(암/항암/당뇨/면역강화 등)
- B2B 거래처(정관장 등) 직접 명시 금지
- 일반식품 표시 가이드 준수

## 적용 순서

1. 카페24 어드민 → 디자인 → 스킨 추가 → 빈 스킨 생성
2. 본 폴더 내용을 FTP/디자인 편집기로 업로드
3. `module="..."` 자리를 어드민의 모듈 추가 마법사로 실제 모듈로 치환
4. 메뉴·카테고리·상품 등록 후 변수(`##...##`) 확인
5. 모바일/PC 스킨 분리 운영 여부는 카페24 설정에 따라 결정(§1.3)
