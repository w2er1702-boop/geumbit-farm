# 금빛농원 카페24 풀커스텀 스킨 (1차 MVP)

설계서 v1.0 기준 카페24 스마트디자인 풀커스텀 스킨. 본 디렉토리를 ZIP으로 묶어 카페24 어드민에 그대로 업로드한다.

## 빠른 배포 (3단계)

```bash
# 1. ZIP 패키지 생성
cd cafe24-skin
./build-zip.sh          # ../geumbit-skin-YYYYMMDD-HHMM.zip 생성

# 2. 카페24 어드민 업로드
#    어드민 → 디자인(쇼핑몰) → 스킨 관리 → 스킨 추가 → ZIP 업로드

# 3. 적용
#    추가한 스킨을 PC몰/모바일몰에 적용 → 디자인 편집 진입
```

## 카페24 스마트디자인 문법 메모

- 각 페이지 최상단:  `<!--@layout(/layout/basic/layout.html)-->`
  → 페이지 본문이 layout.html의 `{$layout_module_content}` 위치에 자동 주입된다.
- 리소스 연결:  `<!--@css(/css/style.css?ver=YYYYMMDD)-->`,  `<!--@js(/js/script.js)-->`
- 부분 포함:  `<!--@import(/layout/basic/_header.html)-->`
- 모듈 블록:  `<div module="product_listnormal">...{$product_name}...</div>`
- 변수:  `{$variable}` 또는 카페24 어드민의 "사용 가능한 변수" 패널 참고
- 캐시 우회:  변경이 즉시 반영되지 않으면 `?ver=20260530` 형태로 버전 파라미터 부여

> 모듈(`module="..."`) 영역의 마크업은 카페24가 자동 반복 렌더링하므로, 실제 운영 시에는 카드/행 1개만 두면 된다. 1차 본 골격에는 시각 확인용 placeholder를 함께 두었으니 어드민 적용 후 제거할 것.

## 상품 데이터 — 1차 확정안 (M1·M2 반영)

| 상품 | 옵션(중량) | 비고 |
|---|---|---|
| 상황버섯 원물 | 30g · 100g · 250g · 500g | 30g·100g은 "참나무" 라인을 원물 옵션으로 통합 |
| 상황버섯 슬라이스 | 250g · 500g | |
| 혼합 세트 | 500g (원물250+슬라이스250) | 선물 추천 |
| 진액 | 90ml × 30포 | 단일 구성 |
| 발효액 | 1박스 | 단일 구성 |

카페24 어드민에서 "옵션 사용"으로 등록하고, 옵션별 가격/재고를 분리한다.

## 어드민 후속 작업 (스킨 업로드 후)

1. **상품 등록** — 위 5개 상품 + 옵션. 이미지·상세 카피는 설계서 §4.4 9단 템플릿대로.
2. **카테고리 생성** — 원물 / 슬라이스 / 혼합세트 / 진액 / 발효액. 헤더 메뉴의 `cate_no=` 값을 실 번호로 맞춘다.
3. **게시판 생성** — 공지사항 / 상품후기 / 1:1문의. 헤더의 `board_no=` 값을 실 번호로 맞춘다.
4. **운영자 정보** — 쇼핑몰 설정 → 내 쇼핑몰 정보. `{$company_name}` 등 푸터 변수가 여기서 채워진다.
5. **소셜로그인** — 회원관리 → 카카오 소셜로그인 활성화.
6. **PG 가맹 심사 신청** — 별도 1~2주 (§7.4).

## 미결 항목 후속 (M3·M4·M5·M6)

| # | 항목 | 카페24 어드민에서의 위치 |
|---|---|---|
| M3 | 도메인 (`ycgoldenfarm.co.kr` 등) | 쇼핑몰 설정 → 도메인 |
| M4 | 배송비 (무료배송 기준, 30g 3,000원) | 주문관리 → 배송 → 배송비 정책 |
| M5 | 로고 국문형 | SkinImg/ 에 업로드 후 `_header.html` 로고 부분 교체 |
| M6 | 촬영 이미지 | 상품 등록 시 업로드. 미준비 분은 placeholder가 자동 노출 |

## 콘텐츠 가이드 (§7)

- 질병명·치료·면역 보장 표현 **금지** (암·당뇨·면역강화 등)
- B2B 거래처(정관장 등) 직접 명시 **금지**
- 전자상거래법 필수 표기 ─ 푸터 변수로 자동 채워짐. 신고번호 미발급 상태면 임시 텍스트로 대체

## 로컬에서 미리보기

카페24 모듈/변수는 서버에서만 치환되므로 로컬에서는 골격·스타일만 검증할 수 있다.

```bash
# 임의의 정적 서버 사용
npx serve cafe24-skin
# → /main/index.html 등을 열어 디자인 토큰·반응형·플로팅바 정도 확인
```
